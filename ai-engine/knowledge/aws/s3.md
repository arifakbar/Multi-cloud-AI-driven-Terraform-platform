# AWS S3 Security Best Practices

## 1. Public Access Block
### Risk
Publicly accessible S3 buckets may expose confidential data.

### CIS Benchmark Reference
CIS AWS 3.2: "Ensure S3 bucket public access block is enabled."

### Required Arguments
To fix public access issues, ensure the `aws_s3_bucket_public_access_block` resource uses:
- `block_public_acls = true`
- `block_public_policy = true`
- `ignore_public_acls = true`
- `restrict_public_buckets = true`

Severity: HIGH

## 2. Server-Side Encryption
### Risk
Unencrypted S3 buckets expose sensitive data at rest.

### Required Arguments
Enable encryption using `server_side_encryption_configuration` block with:
- `sse_algorithm = "AES256"` or `aws:kms`

Severity: HIGH
